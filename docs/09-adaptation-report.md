# Phase 4: 适配实施与报告

## Phase 4-2: 生成适配报告（Phase 4 续）

**定位**：记录实施状态和遗留问题

**前置条件**：
- 适配实施已完成
- 所有修改已验证通过（或失败已处理）

**输入**：
- 用户批准的适配方案
- 实际修改的源码
- 实施状态记录

**输出**：
- `reports/<库名>-adaptation-report.md` - 简化报告（实施状态 + 遗留问题）

**关键原则**：
- ✅ 简洁明了（仅记录实施状态和遗留问题）
- ✅ 记录与方案的偏差
- ✅ 记录遗留问题（反复修改方案都未能解决的）
- ❌ 不重复方案中的详细内容

## AI 执行检查清单

在生成报告前，确认已完成：

- [ ] 所有修改已实施
- [ ] 实施状态已收集
- [ ] 遗留问题已记录（如有）
- [ ] 与方案的偏差已记录（如有）
- [ ] 语法验证已通过
- [ ] 简化报告已生成（`reports/<库名>-adaptation-report.md`）

---

## 报告质量标准

### ✅ 必须包含

1. **实施状态** - 成功/失败统计
2. **遗留问题** - 反复修改方案都未能解决的
3. **偏差记录** - 与方案的差异

### ❌ 避免问题

1. **重复方案** - 不是方案的复制粘贴
2. **隐瞒失败** - 失败必须记录
3. **遗漏偏差** - 方案外修改必须说明
