# Phase 2-1：读取任务表

目标：
- 从任务表中提取本轮需要处理的三方库
- 建立后续克隆与适配所需的基础任务数据

## 输入

- `libs/porting-tasks-YYYY-MM-DD.xlsx`

## 输出

- 待处理库列表
- 每个库的最小元信息：
  - 库名
  - 仓库地址
  - 版本
  - 备注
  - 可选：是否优先尝试 `lycium`

## AI 执行步骤

### 1. 找到当天正式任务表

优先读取当天日期对应文件；若不存在，则读取 `libs/` 中最新的正式任务表。

约束：
- 不允许直接读取 `libs/porting-tasks-模板.xlsx`
- 只有 `porting-tasks-YYYY-MM-DD.xlsx` 这类正式任务表才能作为 Phase 2 输入
- 如果只有模板、没有正式任务表，说明还停留在 Phase 1，必须等待用户补完后再继续

### 2. 读取有效行

只处理至少具备以下字段的行：
- 库名
- Git 仓库

### 3. 规范化字段

建议统一成：
- `lib_name`
- `repo_url`
- `version`
- `status`
- `note`

### 4. 输出任务确认

向用户反馈本轮将处理的库，但不需要在此阶段进入 STOP。

### 5. 标记代码来源模型

AI 应在内部区分两类后续路径：
- `libs/<库名>/`：给 Phase 3/4 分析和 fallback 用
- `HPKBUILD recipe`：给 Phase 5 的 `lycium` 用

## 处理规则

- `版本` 为空：允许继续，后续按仓库默认分支处理
- `状态` 为空：默认视为“待处理”
- `备注` 仅作为辅助信息，不阻塞流程
- 缺少 `库名` 或 `Git 仓库`：才需要用户补充

补充规则：
- 任务表为空，或只有表头没有有效行：中断并提示用户补全，不继续进入代码拉取
